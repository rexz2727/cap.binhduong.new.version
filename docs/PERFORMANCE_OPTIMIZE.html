Bundle Size — Bỏ framer-motion

components/layout/PageTransition.tsx đã được viết lại thành CSS thuần:

"use client";
import { usePathname } from "next/navigation";
export default function PageTransition({ children }: { children: React.ReactNode }) {
  const pathname = usePathname();
  return <div key={pathname} className="animate-fade-in">{children}</div>;
}

animate-fade-in được định nghĩa trong app/globals.css (opacity 0→1, 0.5s ease-out).

safeFetch — Không block render

async function safeFetch<T>(fn: () => Promise<T>, fallback: T): Promise<T> {
  if (!isConfigured) return fallback;
  try { return await fn(); } catch { return fallback; }
}

II. Cơ hội tối ưu tiếp theo

Ưu tiên cao

Rate limiting API routes

✅ Đã triển khai: 5 req/phút/IP cho /api/feedback và /api/qna. Có thể nâng cấp lên Upstash Redis cho chính xác tuyệt đối.

Lazy load sections cuối trang chủ

PhotoAlbumPreview và VideoPreview dùng dynamic import với ssr:false hoặc IntersectionObserver khi cần tối ưu thêm.

Ưu tiên trung bình

Sitemap + robots.txt

✅ Đã có: app/sitemap.ts và app/robots.ts. Google index đúng các trang công khai.

YouTube click-to-play

Thay iframe YouTube tải ngay bằng thumbnail + click để play. Tránh load script YouTube nặng (~500KB).

Ưu tiên thấp

OpenGraph dynamic images

app/opengraph-image.tsx — Hiển thị đẹp khi share lên Zalo/Facebook.

Service Worker / PWA

Cache static assets để tải nhanh hơn trên mạng chậm. Phù hợp nếu đơn vị muốn cài vào màn hình điện thoại.

IV. Quy tắc không được phá vỡ

  • Không thêm animation library (framer-motion, gsap...) — dùng CSS transition
  • Không load ảnh bằng <img> thường — luôn dùng next/image
  • Không import toàn bộ icon library — chỉ import icon cần dùng hoặc SVG inline
  • Không để data fetch trong Client Component nếu có thể làm ở Server Component
  • Không tắt ISR (revalidate: 0 hoặc cache: 'no-store') nếu nội dung không cần real-time
